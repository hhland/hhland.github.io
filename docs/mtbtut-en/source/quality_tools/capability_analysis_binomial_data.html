Data requirements for Binomial Capability Analysis  

What kind of data is required?

Your data must follow the binomial distribution. In other words, your process output must have only two outcomes: a product (or event) is defective or not. The number of defects on a product is not important. For example, a circuit board that has 1, 2, or 10 faulty features counts as 1 defective item.

Your data must contain the number of defective items, as well as the total number of items that you inspected or observed.

Enter the number of defective items for each sample in one column. These counts must be nonnegative integers. For example:
  • Number of delayed flights out of 100 total flights
  • Number of dead batteries out of a batch of 50
  • Number of rejected insurance claims out of 500 submitted

You must also enter the sample size, which equals the total number of events you observed. Sample sizes must be positive integer values. If the sample size varies for each proportion, record the sample sizes in a column. If the sample size is constant, type this value into the dialog box.

What should my worksheet look like?

Enter all observations in a single column, with a second column of sample sizes

If all sample sizes are equal, you can omit the column of sample sizes and enter the sample size in the dialog box.
